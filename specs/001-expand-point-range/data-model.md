# Data Model: Expand Point Range

**Feature**: 001-expand-point-range
**Date**: 2026-02-25

## Entities

### Discipline

Stored in localStorage under key `dd_disciplines` as a JSON array.

| Field     | Type    | Current Rule | New Rule     | Notes                       |
|-----------|---------|--------------|------------- |-----------------------------|
| id        | string  | unique ID    | no change    | Generated by `generateId()` |
| name      | string  | max 60 chars | no change    |                             |
| points    | number  | 1–50         | **1–500**    | Only validation range changes |
| active    | boolean | true/false   | no change    |                             |
| createdAt | string  | YYYY-MM-DD   | no change    |                             |

### Migration

**No migration needed.** The new range (1–500) is a superset of
the old range (1–50). All existing data is valid under the new
rules. The `loadDisciplines()` function sets `points` default
to 5 if undefined, which remains correct.

## Validation Rules

| Surface           | Current              | New                  |
|-------------------|----------------------|----------------------|
| Add modal input   | `min="1" max="50"`   | `min="1" max="500"`  |
| Manage tab input  | `min="1" max="50"`   | `min="1" max="500"`  |
| JS add handler    | clamp to 1–50        | clamp to 1–500       |
| JS manage handler | clamp to 1–50        | clamp to 1–500       |
