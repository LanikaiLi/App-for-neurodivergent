# App Data Model (ERD)

**Last synced**: 2026-03-07 — from feature `002-discipline-scheduling`

All data is persisted in the browser's `localStorage`. There is no backend or database. Four keys are used.

---

## `dd_disciplines` — Discipline list

JSON array of discipline objects.

| Field        | Type           | Required | Description |
|-------------|----------------|----------|-------------|
| `id`        | string         | yes      | Unique identifier (generated by `Date.now().toString(36) + random`) |
| `name`      | string         | yes      | Display name (max 60 chars) |
| `points`    | number         | yes      | Point value (1–500). Default: 5 |
| `active`    | boolean        | yes      | If `true`, included in Today checklist. Default: `true` |
| `createdAt` | string         | yes      | Date discipline was created, format `YYYY-MM-DD` |
| `schedule`  | Schedule \| undefined | no | Scheduling/recurrence info. When missing, treated as `{ recurrence: "daily", preferredTime: null }` |

### Schedule (nested object on Discipline)

| Field           | Type           | Required | Description |
|-----------------|----------------|----------|-------------|
| `recurrence`    | string         | yes      | `"daily"` \| `"weekly"` \| `"custom"` |
| `weekdays`      | number[]       | conditional | 0–6 (Sun–Sat). Required and ≥ 1 element when recurrence is `weekly`, or `custom` with unit `week`. |
| `interval`      | number         | conditional | N ≥ 1. Required when recurrence is `custom`. Cap: 365 (day), 52 (week). |
| `unit`          | string         | conditional | `"day"` \| `"week"`. Required when recurrence is `custom`. |
| `referenceDate` | string         | conditional | `YYYY-MM-DD`. Required when recurrence is `custom`. Defaults to `createdAt`. |
| `preferredTime` | string \| null | yes      | `"HH:MM"` (24h) or `null` for Anytime. |

**Introduced by**: `001-expand-point-range` (points range 1→500), `002-discipline-scheduling` (schedule object).

---

## `dd_records` — Daily completion records

JSON object keyed by date string.

```
{
  "YYYY-MM-DD": {
    "<discipline_id>": true | false,
    ...
  },
  ...
}
```

| Level | Key / Field | Type | Description |
|-------|-------------|------|-------------|
| Root  | `"YYYY-MM-DD"` | object | One entry per day that has any interaction |
| Day   | `"<discipline_id>"` | boolean | `true` if discipline was checked off that day |

**Relationships**: Each discipline id in a day record references a discipline in `dd_disciplines`. Orphaned ids (from deleted disciplines) are cleaned up on delete.

---

## `dd_rewards` — Treats journal

JSON array of reward objects.

| Field         | Type           | Required | Description |
|---------------|----------------|----------|-------------|
| `id`          | string         | yes      | Unique identifier |
| `date`        | string         | yes      | Date spent, format `YYYY-MM-DD` |
| `pointsSpent` | number         | yes      | Points deducted for this treat (≥ 1) |
| `description` | string         | yes      | User description of the treat (max 100 chars) |
| `image`       | string \| null | no       | Base64 data URL of resized photo, or `null` |

**Relationships**: Points balance = total earned (from `dd_records` × discipline points) − total spent (sum of `pointsSpent` in `dd_rewards`).

---

## `dd_exchange` — Exchange rate settings

JSON object (singleton).

| Field  | Type   | Required | Default | Description |
|--------|--------|----------|---------|-------------|
| `rate` | number | yes      | 100     | How many points equal `value` units |
| `value`| number | yes      | 1       | Real-world value per `rate` points |
| `unit` | string | yes      | "dollar" | Currency or unit name (max 20 chars) |

## DBML (for dbdiagram.io)

Copy the block below and paste into [dbdiagram.io](https://dbdiagram.io) to generate the ERD.

```dbml
// Daily Disciplines — App Data Model
// All data persisted in browser localStorage. No backend.
// Modeled as tables for ERD visualization; actual storage is JSON in localStorage.

Table dd_disciplines {
  id varchar [primary key, note: 'Generated by Date.now().toString(36) + random']
  name varchar [not null, note: 'Display name, max 60 chars']
  points integer [not null, default: 5, note: '1–500']
  active boolean [not null, default: true, note: 'If true, included in Today checklist']
  createdAt varchar [not null, note: 'YYYY-MM-DD']
}

Table dd_disciplines_schedule {
  discipline_id varchar [primary key, not null, note: 'FK to dd_disciplines.id']
  recurrence varchar [not null, note: 'daily | weekly | custom']
  weekdays varchar [note: 'Array of 0–6 (Sun–Sat). Required when weekly or custom+week']
  interval integer [note: 'N >= 1. Required when custom. Cap 365 (day) / 52 (week)']
  unit varchar [note: 'day | week. Required when custom']
  referenceDate varchar [note: 'YYYY-MM-DD. Required when custom. Defaults to createdAt']
  preferredTime varchar [note: 'HH:MM (24h) or null for Anytime']
}

Table dd_records {
  date varchar [not null, note: 'YYYY-MM-DD']
  discipline_id varchar [not null, note: 'References dd_disciplines.id']
  completed boolean [not null, note: 'true if checked off that day']
}

Table dd_rewards {
  id varchar [primary key, note: 'Unique identifier']
  date varchar [not null, note: 'YYYY-MM-DD']
  pointsSpent integer [not null, note: '>= 1']
  description varchar [not null, note: 'Max 100 chars']
  image varchar [note: 'Base64 data URL of resized photo, or null']
}

Table dd_exchange {
  rate integer [not null, default: 100, note: 'How many points equal value units']
  value float [not null, default: 1, note: 'Real-world value per rate points']
  unit varchar [not null, default: 'dollar', note: 'Currency or unit name, max 20 chars']
}

// --- Relationships ---

Ref: dd_disciplines_schedule.discipline_id - dd_disciplines.id

Ref: dd_records.discipline_id > dd_disciplines.id
```

## Relationships diagram (text)

```
dd_disciplines[]
  ├── .id ──────────────► dd_records["YYYY-MM-DD"][discipline_id]
  ├── .points ──────────► used to calculate earned points in dd_records
  └── .schedule          (scheduling/recurrence; display-only grouping)

dd_records{}
  └── keyed by date ──► daily completion map { discipline_id: bool }

dd_rewards[]
  └── .pointsSpent ────► deducted from balance (earned − spent)

dd_exchange{}
  └── conversion config (rate, value, unit)
```
